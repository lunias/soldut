# Soldut — credits

This file lists every shipped asset under `assets/` along with its
source, license, and (where AI-assisted) reproducibility metadata.

Steam disclosure (Jan 17 2026 update) requires a generic AI disclosure
on the store page; this file is the canonical source for the disclosure
language.

A build-time script (`tools/credits/check.sh` — TODO) walks `assets/`
and fails CI if any shipped file lacks a row here.

----------------------------------------------------------------------
DISCLOSURE
----------------------------------------------------------------------

Some 2D background, concept, and (potentially) mech-related assets in
this game were generated with the assistance of Stable Diffusion XL
(Illustrious-XL fine-tune, SDXL-derived OpenRAIL-M-equivalent license)
via ComfyUI, with geometry locked by hand-authored/programmatic
skeleton inputs (ControlNet), style anchored to a single approved
reference image (IP-Adapter), and substantial post-generation editing —
2-color palette mapping, halftone screening, nearest-point
downsampling, per-part cropping, compositing, and integration — by the
developer.

The per-chassis mech sprite atlases (`assets/sprites/<chassis>.png`)
are sliced by `tools/comfy/extract_gostek.py` from "gostek part sheets"
— hand-authored (or externally-produced) reference images of the 22
mech body parts, in the layout Soldat's gostek model uses. Whether a
given part sheet was itself AI-generated is recorded in that asset's
row below; the slice + palette-snap + atlas-pack step is deterministic
tooling, not generation.

Each AI-assisted asset is listed below with its provenance and (where
applicable) workflow file, LoRA stack, IP-Adapter anchor, seed, and
modification chain. The same metadata is mirrored in
`documents/art_log.md` per the project's mandatory iteration-log policy
(see documents/m5/11-art-direction.md §"License hygiene").

Hand-drawn, traced, and CC0-sourced assets are listed alongside; their
source paths and modifications are recorded the same way.

assets/sprites/trooper.png  (P15 revised — gostek part-sheet extraction)
  source:           tools/comfy/gostek_part_sheets/trooper_gostek_v1.png
                    (a "gostek part sheet" — one 2048×2048 reference image
                    with all 22 mech body parts laid out flat-shaded,
                    black-outlined, and captioned in reading order, in the
                    layout Soldat's gostek model uses for per-part rigid
                    skinning; supplied by the team)
  sheet_provenance: team-authored reference sheet; the production method
                    (hand-drawn / vector / traced / generated out-of-repo)
                    is not recorded in this commit. If the sheet was derived
                    from a licensed source (e.g. an AI model + LoRA, or a
                    traced image), record that here and in documents/art_log.md.
  pipeline:         tools/comfy/extract_gostek.py
                    tools/comfy/gostek_part_sheets/trooper_gostek_v1.png
                    trooper --palette foundry
                    (deterministic: scipy connected-component detect of the
                    22 part shapes → reading-row order → s_default_parts slot
                    → per-part rotate/flip [forearms ROTATE_270, head
                    FLIP_TOP_BOTTOM] → resize to runtime sprite size → border-
                    flood white-key → composite at the dst rects that mirror
                    src/mech_sprites.c::s_default_parts. No transcribe step.)
  modification:     2-colour palette snap to Foundry (dark #1A1612 + accent
                    #D8731A) via tools/comfy/crop_canonical.post_process_tile,
                    + nearest-neighbor half-size round-trip per tile. No
                    Bayer/halftone is baked in — the runtime halftone_post
                    shader screens at framebuffer resolution.
  manifest:         tools/comfy/gostek_part_sheets/trooper_gostek_manifest.txt
                    (per-part src bbox → dst, written by extract_gostek.py;
                    detected 22 part shapes, 7 reading rows 3/2/6/2/2/2/5)
  loras:            n/a (no diffusion step in this path)
  ip_adapter:       n/a
  seed:             n/a
  human_direction:  gostek part sheet authored/sourced; extractor's slot/
                    rotation table (SLOT_FIX) tuned against the preview;
                    palette authored; in-game shot-test verified (Trooper
                    renders as a chunky industrial mech, not capsules).
  caveats:          first gostek pass — a clean, recognizable industrial mech
                    in the Foundry 2-colour register. Refinement of the sheet
                    itself (proportions, panel detail) is the iterative lever
                    in P16; the renderer/extractor side is locked.
  hardware_note:    NOT a build prerequisite — the extractor needs Python +
                    PIL + numpy + scipy (the ComfyUI venv has them). To
                    regenerate: `python tools/comfy/extract_gostek.py
                    tools/comfy/gostek_part_sheets/trooper_gostek_v1.png
                    trooper --palette foundry`.
  diffusion_alt:    The AI-diffusion path (tools/comfy/workflows/soldut/
                    mech_chassis_canonical_v1.json full Pipeline 1 + _8gb_v1
                    subset; asset.py orchestrator; crop_canonical.py canonical-
                    crop mode) stays in the repo but is NOT what shipped here —
                    it plateaued below the art bar (see TRADE_OFFS.md "Chassis
                    art is hand-authored gostek part sheets, not the
                    AI-diffusion pipeline"). The previously-shipped Trooper
                    atlas from that path (mech_chassis_canonical_v1.json,
                    Illustrious-XL + ControlNet-Union promax + IP-Adapter Plus,
                    seed 1000001) is superseded by this row.
